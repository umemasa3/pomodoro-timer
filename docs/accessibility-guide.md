## 概要

ポモドーロタイマーは、すべてのユーザーが平等にアクセスできるよう設計されています。本ガイドでは、障害のある方や特別なニーズを持つ方が本アプリケーションを効果的に利用するための情報を提供します。

### アクセシビリティへの取り組み

- **WCAG 2.1 AA準拠**: Web Content Accessibility Guidelines 2.1のAA基準に準拠
- **継続的改善**: ユーザーフィードバックに基づく継続的な改善
- **包括的設計**: 多様なユーザーのニーズを考慮した設計
- **技術的対応**: 支援技術との互換性を重視

## 対応状況

### 準拠基準

| 基準              | レベル | 対応状況    |
| ----------------- | ------ | ----------- |
| WCAG 2.1          | A      | ✅ 完全対応 |
| WCAG 2.1          | AA     | ✅ 完全対応 |
| WCAG 2.1          | AAA    | 🔄 部分対応 |
| JIS X 8341-3:2016 | AA     | ✅ 完全対応 |
| Section 508       | -      | ✅ 完全対応 |

### 支援技術対応

| 技術               | 対応状況 | 備考                            |
| ------------------ | -------- | ------------------------------- |
| スクリーンリーダー | ✅ 対応  | NVDA, JAWS, VoiceOver, TalkBack |
| 音声認識           | ✅ 対応  | Dragon NaturallySpeaking等      |
| スイッチ操作       | ✅ 対応  | キーボードナビゲーション経由    |
| 拡大鏡             | ✅ 対応  | 高コントラストモード対応        |

## キーボード操作

### 基本操作

すべての機能はキーボードのみで操作可能です。

#### グローバルショートカット

| キー          | 機能                                     |
| ------------- | ---------------------------------------- |
| `Tab`         | 次の要素にフォーカス移動                 |
| `Shift + Tab` | 前の要素にフォーカス移動                 |
| `Enter`       | ボタンの実行、リンクの選択               |
| `Space`       | チェックボックスの切り替え、ボタンの実行 |
| `Esc`         | ダイアログの閉じる、操作のキャンセル     |
| `Arrow Keys`  | リスト内の移動、メニュー操作             |

#### タイマー操作

| キー    | 機能                       |
| ------- | -------------------------- |
| `Space` | タイマーの開始/一時停止    |
| `R`     | タイマーのリセット         |
| `S`     | 次のセッションにスキップ   |
| `T`     | タスク選択ダイアログを開く |

#### ナビゲーション

| キー | 機能                 |
| ---- | -------------------- |
| `1`  | タイマーページに移動 |
| `2`  | タスクページに移動   |
| `3`  | 統計ページに移動     |
| `4`  | 設定ページに移動     |
| `H`  | ヘルプを表示         |

### フォーカス管理

- **視覚的フォーカス**: すべてのフォーカス可能要素に明確な視覚的インジケーター
- **論理的順序**: 画面上の論理的な順序でフォーカスが移動
- **フォーカストラップ**: モーダルダイアログ内でフォーカスが適切に管理
- **スキップリンク**: メインコンテンツへの直接移動が可能

### カスタムキーボードショートカット

設定画面からキーボードショートカットをカスタマイズできます：

1. 設定 > アクセシビリティ > キーボードショートカット
2. 変更したいショートカットを選択
3. 新しいキーの組み合わせを入力
4. 保存して適用

## スクリーンリーダー対応

### 対応スクリーンリーダー

- **Windows**: NVDA, JAWS, Windows ナレーター
- **macOS**: VoiceOver
- **iOS**: VoiceOver
- **Android**: TalkBack
- **Linux**: Orca

### ARIA属性の活用

- **ランドマーク**: `main`, `navigation`, `banner`, `contentinfo`
- **ライブリージョン**: タイマー状態の変更を自動通知
- **状態情報**: ボタンの状態、進捗状況を音声で通知
- **説明**: 複雑な操作に対する詳細な説明

### 音声フィードバック

#### タイマー関連

- セッション開始時: "25分の集中時間を開始しました"
- セッション終了時: "集中時間が終了しました。5分の休憩を開始してください"
- 残り時間: 設定により定期的に残り時間を通知

#### 操作フィードバック

- ボタン押下: "タイマーを開始しました"
- 設定変更: "作業時間を30分に変更しました"
- エラー: "入力内容に誤りがあります。タスク名を入力してください"

### スクリーンリーダー設定

1. **詳細度の調整**
   - 設定 > アクセシビリティ > スクリーンリーダー
   - 通知の詳細度を選択（簡潔/標準/詳細）

2. **自動読み上げ**
   - タイマー状態の変更を自動通知
   - 重要な通知の優先読み上げ

## 視覚的アクセシビリティ

### 色とコントラスト

#### 高コントラストモード

- **有効化**: 設定 > アクセシビリティ > 高コントラストモード
- **コントラスト比**: 最低4.5:1（AA基準）、推奨7:1（AAA基準）
- **色に依存しない設計**: 色以外の手段でも情報を伝達

#### カラーパレット

- **ダークモード**: 目の疲労軽減
- **ライトモード**: 明るい環境での視認性
- **カスタムテーマ**: 個人の視覚ニーズに合わせた調整

### 文字とフォント

#### フォントサイズ

- **標準**: 16px（ブラウザ標準）
- **大**: 18px
- **特大**: 20px
- **最大**: 24px

#### フォントファミリー

- **システムフォント**: OS標準フォントを使用
- **読みやすさ**: 判読しやすいサンセリフフォント
- **カスタムフォント**: ディスレクシア対応フォントオプション

### 視覚的インジケーター

- **フォーカス**: 明確な枠線とハイライト
- **ホバー**: マウスオーバー時の視覚的フィードバック
- **アクティブ**: 押下時の視覚的フィードバック
- **無効**: 無効な要素の明確な表示

### 拡大・ズーム対応

- **ブラウザズーム**: 400%まで対応
- **レスポンシブデザイン**: 拡大時のレイアウト崩れなし
- **スクロール**: 横スクロールが発生しない設計

## 聴覚的アクセシビリティ

### 音声通知の代替手段

#### 視覚的通知

- **画面フラッシュ**: セッション終了時の画面点滅
- **通知バナー**: 画面上部の通知表示
- **プログレスバー**: 視覚的な進捗表示

#### 振動通知（モバイル）

- **セッション終了**: 振動パターンで通知
- **重要な通知**: 異なる振動パターンで区別

### 音声設定

1. **音声通知の無効化**
   - 設定 > 通知 > 音声通知をオフ

2. **視覚的通知の有効化**
   - 設定 > アクセシビリティ > 視覚的通知をオン

3. **字幕・テキスト表示**
   - 音声内容をテキストで表示

## 認知的アクセシビリティ

### シンプルなインターフェース

- **一貫性**: 全画面で統一されたデザインパターン
- **予測可能性**: 標準的なUI要素の使用
- **明確な階層**: 情報の重要度に応じた視覚的階層
- **最小限の認知負荷**: 必要な情報のみを表示

### エラー防止と回復

#### エラー防止

- **入力検証**: リアルタイムでの入力チェック
- **確認ダイアログ**: 重要な操作前の確認
- **自動保存**: データの自動保存機能

#### エラー回復

- **明確なエラーメッセージ**: 問題と解決方法を明示
- **元に戻す機能**: 操作の取り消し
- **ヘルプの提供**: 各画面でのコンテキストヘルプ

### 時間制限への配慮

- **時間延長**: セッション時間の柔軟な調整
- **一時停止**: いつでもタイマーを一時停止可能
- **自動保存**: 時間切れによるデータ損失の防止

### 記憶支援

- **進捗の可視化**: 現在の状況を常に表示
- **履歴の保持**: 過去の作業記録を確認可能
- **リマインダー**: 重要な操作のリマインド機能

## モバイルアクセシビリティ

### タッチ操作

#### タッチターゲット

- **最小サイズ**: 44px × 44px（iOS）、48dp × 48dp（Android）
- **適切な間隔**: タッチターゲット間の十分な間隔
- **誤操作防止**: 重要なボタンの配置に配慮

#### ジェスチャー

- **代替手段**: すべてのジェスチャーに代替操作を提供
- **シンプルなジェスチャー**: 複雑なジェスチャーを避ける
- **カスタマイズ**: ジェスチャーの有効/無効を選択可能

### モバイル固有の支援技術

#### iOS

- **VoiceOver**: 完全対応
- **Switch Control**: キーボードナビゲーション経由で対応
- **Voice Control**: 音声コマンドでの操作

#### Android

- **TalkBack**: 完全対応
- **Select to Speak**: テキスト選択での読み上げ
- **Live Caption**: 音声の自動字幕表示

### 画面の向き

- **縦向き**: 標準的な使用方法
- **横向き**: 対応済み、レイアウト自動調整
- **固定**: 必要に応じて画面向きを固定可能

## 設定とカスタマイズ

### アクセシビリティ設定画面

設定 > アクセシビリティから以下の設定が可能です：

#### 視覚設定

- [ ] 高コントラストモード
- [ ] 大きなフォント
- [ ] アニメーションの削減
- [ ] 色覚異常対応

#### 聴覚設定

- [ ] 音声通知の無効化
- [ ] 視覚的通知の有効化
- [ ] 振動通知（モバイル）

#### 操作設定

- [ ] キーボードナビゲーション
- [ ] カスタムショートカット
- [ ] タッチ操作の調整
- [ ] 自動フォーカス

#### 認知支援

- [ ] シンプルモード
- [ ] 詳細な説明表示
- [ ] 操作確認ダイアログ
- [ ] 自動保存間隔

### プリセット設定

よく使用される設定の組み合わせをプリセットとして提供：

1. **視覚障害者向け**
   - 高コントラストモード
   - 大きなフォント
   - スクリーンリーダー最適化

2. **聴覚障害者向け**
   - 音声通知無効
   - 視覚的通知有効
   - 振動通知有効

3. **認知障害者向け**
   - シンプルモード
   - 詳細な説明表示
   - 操作確認ダイアログ

4. **運動障害者向け**
   - キーボードナビゲーション
   - 大きなタッチターゲット
   - 長押し時間の調整

## サポートとフィードバック

### アクセシビリティサポート

#### 問い合わせ方法

1. **アプリ内フィードバック**
   - 設定 > ヘルプ・サポート > アクセシビリティフィードバック

2. **メール**
   - 件名に「アクセシビリティ」を記載

#### 対応時間

- **通常の問い合わせ**: 3営業日以内
- **緊急のアクセシビリティ問題**: 24時間以内

### フィードバックの活用

ユーザーからのフィードバックは以下のように活用されます：

1. **即座の対応**: 重大な問題の迅速な修正
2. **機能改善**: 定期アップデートでの改善実装
3. **新機能開発**: ユーザーニーズに基づく新機能開発

### コミュニティ

- **ユーザーグループ**: アクセシビリティユーザーのコミュニティ
- **ベータテスト**: 新機能のアクセシビリティテスト参加
- **フィードバック共有**: 他のユーザーとの経験共有

## 継続的改善

### 定期的な評価

- **月次**: アクセシビリティ指標の確認
- **四半期**: 外部専門家による評価
- **年次**: 包括的なアクセシビリティ監査

### 技術的改善

- **最新基準への対応**: WCAG等の最新基準への準拠
- **新技術の採用**: 支援技術の進歩への対応
- **パフォーマンス最適化**: 支援技術での快適な利用

### ユーザー参加

- **ユーザビリティテスト**: 障害のあるユーザーによるテスト
- **フィードバック収集**: 定期的なアンケート調査
- **共同開発**: ユーザーとの協力による機能開発
